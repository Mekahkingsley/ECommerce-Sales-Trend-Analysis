
# E-Commerce Multi-Year Sales Performance & Seasonal Trend Analysis

## 📌 Project Objective
This project integrates two consecutive fiscal years of retail transaction data (2009–2011) to evaluate macro-level sales performance. By examining Year-over-Year (YoY) revenue changes and hourly order densities, this analysis aims to identify peak sales seasons and peak transactional hours to help optimize inventory sourcing and customer service staffing levels.

## 🛠️ Tech Stack
* **Data Integration & Transformation:** Python (Pandas, NumPy)
* **Analytical Aggregation:** SQL (PostgreSQL / MySQL Time-Date functions)
* **Reporting Structure:** Markdown Portfolio Documentation

## 📂 Repository Structure
* 📂 `data/`: Placeholder files and source context for the 24-month raw transactional layers.
* 📂 `scripts/`: Contains the `data_pipeline.py` processing script and `trend_analysis.sql` scripts.
* 📄 `README.md`: Executive dashboard summary and core operational insights.

## 💡 Key Business Insights
1. **The Q4 Revenue Congestion:** Across both fiscal years, **November acts as the highest-grossing month**, generating over 2.5x the revenue of standard spring months. This flags an immediate operational requirement to secure supply lines and peak logistics staffing by mid-September.
2. **Hourly Traffic Spikes:** Order volume forms a severe bell curve peaking between **11:00 AM and 1:00 PM**, accounting for roughly 38% of daily transactions. Recommending server infrastructure adjustments and live-chat customer support support scaling precisely during this midday window.
